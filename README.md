# YT Bookmarks

Lightweight Chrome extension to bookmark YouTube videos with timestamps.

## Features

- Save current video timestamp with one click
- Organize bookmarks with custom descriptions
- Jump to saved timestamps instantly
- Clean, modern popup interface

## Project Structure
```

├── src/
│ ├── background/ # Service worker (message handling)
│ ├── content/ # YouTube page integration
│ ├── popup/ # Extension popup UI
│ ├── core/ # Bookmark logic & storage
│ ├── types/ # TypeScript definitions
│ └── utils/ # Helper functions
├── assets/ # Icons and images
├── manifest.json # Extension configuration
└── styles.css # Popup styles

````

## Files Explained

### Core Files

**`manifest.json`**
- Extension configuration and metadata
- Declares permissions (`storage`, `tabs`)
- Registers service worker, content scripts, and popup
- Defines web-accessible resources (icons)

**`styles.css`**
- Global styles for popup and content script elements
- CSS variables for theming
- Responsive design and animations

### Source Files (`src/`)

**`background/index.ts`**
- Service worker that runs in the background
- Message router handling bookmark operations (ADD, GET, UPDATE, DELETE)
- Listens for tab updates to notify content script of new videos
- No direct DOM access

**`content/index.ts`**
- Injected into YouTube pages
- Creates and injects the bookmark button on video player
- Captures video metadata (title, channel, timestamp)
- Communicates with background script via messages
- Shows toast notifications for user feedback

**`popup/index.ts`**
- Controls the extension popup UI
- Fetches and displays bookmarks for current video
- Handles user interactions (edit, play, delete)
- Sends messages to background and content scripts

**`core/bookmark.ts`**
- Defines `Bookmark` interface/type
- `createBookmark()` - Factory function to create bookmark objects
- Generates unique IDs using `crypto.randomUUID()`
- Normalizes bookmark data structure

**`core/storage.ts`**
- Abstraction layer over Chrome Storage API
- CRUD operations: `getBookmarks`, `addBookmark`, `updateBookmark`, `deleteBookmark`
- Uses `video:${videoId}` as storage keys
- Handles sorting and data persistence
- `getAllVideos()` for future "all bookmarks" view

**`types/messages.ts`**
- TypeScript type definitions for inter-script messages
- Defines message types and payloads
- Ensures type safety across background, content, and popup

**`utils/index.ts`**
- Helper utilities used across the extension
- `getVideoIdFromUrl()` - Extracts YouTube video ID from URL
- `formatTime()` - Converts seconds to HH:MM:SS format
- `getActiveTabURL()` - Gets current active tab information

### Build Output (`dist/`)

Generated by esbuild during build process:
- `dist/background/index.js` - Bundled service worker
- `dist/content/index.js` - Bundled content script
- `dist/popup/index.js` - Bundled popup script

### Assets (`assets/`)

4. Select the project folder

## How It Works

1. **Content script** (`content/`) injects a bookmark button on YouTube
2. **Background service worker** (`background/`) handles messages and storage
3. **Popup** (`popup/`) displays saved bookmarks for the current video
4. **Storage** (`core/storage.ts`) persists bookmarks using Chrome storage API

### Message Flow

```
YouTube Page → Content Script → Background Script → Storage
                     ↓                    ↓
                  Popup UI ← ← ← ← ← ← ← ←
```

**Example: Adding a bookmark**

1. User clicks bookmark button on YouTube
2. `content/index.ts` captures video metadata
3. Sends `ADD_BOOKMARK` message to `background/index.ts`
4. Background calls `core/storage.ts` to persist
5. Returns updated bookmarks to content script
6. Content script shows success toast
